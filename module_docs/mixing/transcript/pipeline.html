mixing.transcript.pipeline

High-level filler-removal pipeline.

Orchestrates: extract audio -> Scribe transcribe -> filler detection -> ffmpeg cut -> write transcript outputs.

mixing.transcript.pipeline.remove_fillers(input_media: str | Path, output_dir: str | Path, *, output_media: str | Path | None = None, fillers: Iterable[str] = frozenset({'ah', 'eh', 'er', 'erm', 'hmm', 'mhm', 'mm', 'mmm', 'uh', 'uhh', 'uhm', 'um', 'umm'}), audio_events: Iterable[str] = frozenset({'(coughs)'}), api_key: str | None = None, scribe_kwargs: Mapping[str, object] | None = None, apply_keeps_kwargs: Mapping[str, object] | None = None, keep_intermediate: bool = True, scribe_data: dict | None = None) → FillerRemovalResult[source]

End-to-end filler removal.

Parameters:

  • input_media – Video or audio file to clean.

  • output_dir – Directory to write transcripts and intermediate files into. Created if missing.

  • output_media – Output cleaned media path. Defaults to output_dir / f"{stem}.cleaned.mp4".

  • audio_events (fillers /) – Override the default filler / event sets.

  • api_key – ElevenLabs API key (else env ELEVENLABS_API_KEY).

  • scribe_kwargs – Extra kwargs forwarded to transcribe().

  • apply_keeps_kwargs – Extra kwargs forwarded to apply_keeps().

  • keep_intermediate – If False, delete the extracted audio file.

  • scribe_data – If provided, skip the network call and use this dict (must match Scribe’s response shape). Useful for tests / replays.

Returns:

A FillerRemovalResult with file paths and the computed cut/keep ranges.
